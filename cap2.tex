\subsection{Multiplexação PDH}

O termo multiplexação define a tarefa de enviar dois ou mais sinais utilizando o mesmo meio de transmissão. Isso pode ser obtido de diversas maneiras, como, por exemplo, divisão do tempo de acesso ao meio em turnos para cada usuário transmitir.

Segundo \cite{Ramaswami2010}, o PDH utiliza uma multiplexação assíncrona, na qual cada terminal tem seu próprio \textit{clock}, levando a imprecisões entre as suas taxas de funcionamento, mesmo quando adotada um \textit{clock} nominal. Assim, para que haja correção nessas pequenas diferenças, deve ser inserir bits nos fluxos. Para que seja possível se captar um fluxo mais lento de informação multiplexado num fluxo mais rápido, é necessário uma estrutura chamada "montanha de multiplexadores" (ver Figura \ref{fig:muxPDH}.a), o que eleva o custo e complexidade das redes que usam esse tipo. \section{Relacionamento entre Camadas}
\label{sec:layers}
Nesta seção será apresentado o relacionamento transversal e longitudinal entre as camadas de redes.
De modo geral, para explicar o relacionamento longitudinal, o modelo OSI é apresentado, visto na
Figura~\ref{fig:modelo_OSI}. De acordo com \cite{DIOGENES}, a arquitetura do modelo OSI é dividida em 7 níveis,
compreendendo do nível do usuário (Apresentação) até o nível de transmissão do pulso elétrico (Física).

\begin{figure}[H]
	\centering
		\includegraphics[scale=.7]{modelo_OSI.png}
	\caption{Modelo OSI. Fonte:(Cisco NETWORKING ACADEMY PROGRAM, 10 jun 2007).}
	\label{fig:modelo_OSI}
\end{figure}

No \textit{host} de origem, a camada de aplicação, responsável pelo interfaceamento do usuário com a rede, encapsula seus dados para camada de apresentação. Esta camada traduz (codifica) os dados da aplicação para a camada de Sessão, esta responsável por controlar a comunicação entre os host. Os dados da camada de Sessão são encapsulados dentro dos pacotes da camada de Transporte, que tem como função garantir a confiabilidade da transmissão fim-a-fim. Após isso, é passado a camada de Rede, que tem como função o gerenciamento dos melhores caminhos de dados a ser enviados pela rede. A camada de dados, tem com função traduzir os dados provenientes da camada de transporte, e converter em bit, e entrega à camada Física, que é o próprio meio (fibra ótica, cabos coaxial, par trançado, etc.).

Após envio dos dados para camada Física, o processo de interfaceamento entre camada é feito de modo inverso no \textit{host} de destino, onde que cada camada superior, desencapsula os dados da camada imediatamente inferior. Sendo que cada nível de camada no \textit{host} de origem, se comunica apenas com a camada de mesmo nível no \textit{host} de destino.
